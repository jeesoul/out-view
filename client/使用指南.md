# outView 使用指南 - 让 A 电脑远程连接 B 电脑

## 场景说明

```
┌─────────────────┐         ┌─────────────────┐         ┌─────────────────┐
│   电脑 A        │         │  公网服务器      │         │   电脑 B        │
│   (外出电脑)     │  RDP    │  12.45.22.1     │  隧道   │   (家庭电脑)     │
│                 │ ──────> │                 │ <────── │                 │
│  运行 mstsc     │         │  运行服务端      │         │  运行客户端      │
└─────────────────┘         └─────────────────┘         └─────────────────┘
```

**核心原理**：
- 电脑 B 运行客户端，主动连接公网服务器建立隧道
- 电脑 A 通过远程桌面连接到公网服务器的某个端口
- 服务器将数据转发到电脑 B 的 RDP 服务

## 第一步：部署服务端（在公网服务器 12.45.22.1）

### 1.1 上传服务端程序

将 `outview-server.jar` 上传到服务器：
```bash
# 方式一：使用 scp
scp target/outview-server.jar root@12.45.22.1:/opt/outview/

# 方式二：使用 FTP 工具（如 FileZilla）上传
```

### 1.2 安装 Java 环境
```bash
# CentOS/RHEL
yum install java-1.8.0-openjdk -y

# Ubuntu/Debian
apt install openjdk-8-jdk -y
```

### 1.3 启动服务端
```bash
# 前台运行（测试用）
java -jar /opt/outview/outview-server.jar

# 后台运行
nohup java -jar /opt/outview/outview-server.jar > /opt/outview/app.log 2>&1 &
```

### 1.4 开放防火墙端口
```bash
# CentOS/RHEL (firewalld)
firewall-cmd --permanent --add-port=8080/tcp   # 管理后台
firewall-cmd --permanent --add-port=7000/tcp   # 客户端连接
firewall-cmd --permanent --add-port=6000-6500/tcp  # 数据端口
firewall-cmd --reload

# Ubuntu/Debian (ufw)
ufw allow 8080/tcp
ufw allow 7000/tcp
ufw allow 6000:6500/tcp
```

### 1.5 验证服务端

浏览器访问：`http://12.45.22.1:8080/health`

应返回：`{"status":"UP",...}`

## 第二步：生成 Token（在公网服务器管理后台）

### 2.1 访问管理后台
浏览器打开：`http://12.45.22.1:8080/index.html`

### 2.2 生成 Token
1. 点击"生成新 Token"按钮
2. 记录返回的 `deviceId` 和 `token`，例如：
   ```
   deviceId: a1b2c3d4e5f6g7h8
   token: x9y8z7w6v5u4t3s2r1q0
   ```

## 第三步：编译客户端（在当前电脑）

### 3.1 安装 Go

双击运行 `D:\claudeCodeSpace\java\out-view\client\go1.21.6.windows-amd64.msi`

安装完成后，打开新的命令行窗口验证：
```cmd
go version
```

### 3.2 编译客户端

```cmd
cd D:\claudeCodeSpace\java\out-view\client

# 安装依赖
go mod tidy

# 编译 CLI 版本（推荐，体积小）
set CGO_ENABLED=0
go build -ldflags "-s -w" -o outview-client.exe ./cmd/outview-client

# 或编译 GUI 版本（需要 CGO）
build-gui.bat
```

编译成功后会生成 `outview-client.exe`

## 第四步：在电脑 B 运行客户端

### 4.1 复制客户端到电脑 B

将 `outview-client.exe` 复制到电脑 B（家庭电脑）

### 4.2 运行客户端

**方式一：命令行运行**
```cmd
outview-client.exe -host 12.45.22.1 -port 7000 -device-id a1b2c3d4e5f6g7h8 -token x9y8z7w6v5u4t3s2r1q0
```

**方式二：创建配置文件 `config.txt`**
```
host=12.45.22.1
port=7000
device-id=a1b2c3d4e5f6g7h8
token=x9y8z7w6v5u4t3s2r1q0
```

### 4.3 确认连接成功

看到类似输出表示成功：
```
====================================
       outView Client v1.0.0
====================================
Server:      12.45.22.1:7000
Device ID:   a1b2c3d4e5f6g7h8
Local Port:  3389
====================================

[Info] Connecting to server...
[State] Connecting -> Connected
[Register] Success! External port: 6001
[Info] You can now connect to 12.45.22.1:6001 for RDP access
```

**重要：记住分配的端口号（如 6001）！**

---

## 第五步：在电脑 A 远程连接

### 5.1 打开远程桌面连接

按 `Win + R`，输入 `mstsc`，回车

### 5.2 输入连接地址

在"计算机"栏输入：`12.45.22.1:6001`

（6001 是分配给电脑 B 的外部端口）

### 5.3 输入凭据

输入电脑 B 的 Windows 用户名和密码

## 完整流程图

```
1. 公网服务器 (12.45.22.1)
   └── 运行: java -jar outview-server.jar
   └── 开放端口: 8080, 7000, 6000-6500

2. 电脑 B (家庭电脑，要被远程控制的电脑)
   └── 确保开启了远程桌面 (系统属性 → 远程 → 允许远程连接)
   └── 运行: outview-client.exe -host 12.45.22.1 -port 7000 -device-id xxx -token xxx
   └── 记录显示的外部端口 (如 6001)

3. 电脑 A (外出电脑)
   └── 打开 mstsc
   └── 连接: 12.45.22.1:6001
   └── 输入电脑 B 的用户名和密码
```

## 常见问题

### Q1: 客户端显示"连接失败"？
- 检查公网服务器是否启动
- 检查防火墙是否开放 7000 端口
- 检查 deviceId 和 token 是否正确

### Q2: 远程桌面连接不上？
- 确认客户端显示"注册成功"
- 确认使用正确的端口（如 6001，不是 7000）
- 检查防火墙是否开放 6000-6500 端口

### Q3: 连接后黑屏或断开？
- 确认电脑 B 开启了远程桌面功能
- 确认电脑 B 的 RDP 服务正常运行（默认端口 3389）
- 检查电脑 B 的防火墙是否允许 3389 入站

### Q4: 多台电脑怎么办？
- 每台电脑使用不同的 deviceId
- 每台电脑会分配不同的外部端口
- 通过不同端口区分连接到哪台电脑

## 端口说明

| 端口 | 用途 | 谁连接 |
|------|------|--------|
| 8080 | 管理后台 | 管理员浏览器 |
| 7000 | 控制端口 | 电脑 B 客户端 |
| 6000-6500 | 数据端口 | 电脑 A 远程桌面 |

每个客户端连接后会分配一个数据端口，例如：
- 电脑 B-1 分配端口 6001
- 电脑 B-2 分配端口 6002
- ...
